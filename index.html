How DropZone works

Peer-to-Peer

Files travel directly between browsers using WebRTC. A lightweight signaling server helps you find each other, then steps aside. Your data never touches the cloud.

Security modes

Standard uses WebRTC's built-in DTLS encryption—secure for most transfers with zero friction. AES-256-GCM adds an additional end-to-end layer. The key lives only in your URL hash and is never sent to the server.

Adaptive chunks

Instead of guessing a chunk size, DropZone measures your connection in real time and adjusts packet size and window automatically. This keeps memory low and speed high.

Large files

A Service Worker streams incoming data straight to disk. You can receive multi-gigabyte files without running out of RAM.

Settings

Standard is fastest and requires no key management. AES adds a second encryption layer using a key shared via the URL hash.

Adaptive adjusts chunk size and window based on real-time latency. Fixed is useful for testing specific network conditions.

Auto-download

Automatically save files when a transfer completes. Browsers may block multiple pop-ups.
