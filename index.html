How it works

DropZone sends data directly between browsers.

Peer-to-peer

Files travel directly via WebRTC. A signaling server helps you find each other, then steps aside. Data never touches a server.

Two security modes

Standard uses WebRTC's built-in DTLS encryption. AES-256-GCM adds a second end-to-end layer with a key in your URL hash.

Adaptive chunking

DropZone measures latency in real time and adjusts packet size automatically. Transfers stay fast without exhausting memory.

Large files

Transfers complete in chunks and assemble into a downloadable blob. Multi-gigabyte files are supported within your browser's limits.
