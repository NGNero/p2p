How DropZone works

Peer-to-peer — no cloud

Files travel directly between browsers using WebRTC. A lightweight signaling server helps you find each other, then steps aside. Your data never touches a server.

Two security modes

Standard uses WebRTC's built-in DTLS encryption — secure for most transfers. AES-256-GCM adds a second end-to-end layer. The key exists only in your URL hash and is never sent to any server.

Adaptive chunking

DropZone measures round-trip latency in real time and adjusts packet size and send window automatically. Memory stays low and speed stays high.

Streaming large files

A Service Worker streams incoming data directly to disk. Multi-gigabyte transfers complete without exhausting RAM.

Connecting with a key

Go to Receive and paste the full share link, or enter the peer ID and optional AES key separately in the manual fields. This is useful if you received the link and key through different channels for extra security.

Settings

Standard is fastest. AES adds a second encryption layer via a key in the URL hash — never sent to any server.

Adaptive tunes chunk size and window size based on live RTT. Fixed modes are useful for testing specific conditions.

Auto-download

Automatically save files when a transfer completes. Browsers may block multiple simultaneous pop-ups.
